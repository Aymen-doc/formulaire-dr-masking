                Exemples de Règles Métier
            


            

                
                

                    

                        

                            RG03 – Code Postal
                        

                    

                    

                        

                            

                                
Configuration YAML

                                


rule_id: RG03_POSTAL
when: "row['code_postal'] and row['departement']"
action:
  plugin: LookupRef
  field_in: code_postal
  lookup_table: REF_GEO_CP
  lookup_key: row['departement']
exceptions: ["00000", "99999", "88888"]
on_exception: keep_source
on_lookup_fail: default
default: "99999"
priority: 10

                                

                            

                            

                                
Explication

                                

                                    
Objectif : Masquer les codes postaux tout en préservant la cohérence géographique

                                    
Méthode : Utilise une table de référence pour mapper département → codes postaux valides

                                    
Exceptions : Codes spéciaux (00000, 99999, 88888) restent inchangés

                                    
Fallback : Si échec de lookup, utilise "99999" par défaut

                                

                                

                                    
Cas d'usage : Données médicales, enquêtes sociales, études démographiques

                                

                            

                        

                    

                


                
                

                    

                        

                            RG28 – Date Début Soin
                        

                    

                    

                        

                            

                                
Configuration YAML

                                


rule_id: RG28_DDS
when: "row['date_debut_soin']"
action:
  plugin: CalcFunction
  field_in: date_debut_soin
  function: rewrite_dds
priority: 20

                                

                                
Fonction Python

                                


def rewrite_dds(row):
    dds = row['date_debut_soin']
    if dds in {'0001-01-01','1600-01-01','9999-12-31'}:
        return dds
    if is_bank_holiday(dds) or dds.weekday() in (5,6):
        return previous_weekday(dds, target=4)
    if row.get('date_fin_soin'):
        dur = (row['date_fin_soin'] - dds).days + 1
        if dur <= 0:
            return dds + timedelta(days=1)
    return dds

                                

                            

                            

                                
Logique Métier

                                

                                    
1. Dates spéciales : Les dates système (0001-01-01, 1600-01-01, 9999-12-31) restent inchangées

                                    
2. Week-ends et fériés : Décalées au vendredi précédent pour réalisme médical

                                    
3. Cohérence temporelle : Vérifie que date_debut < date_fin

                                    
4. Correction automatique : Si durée ≤ 0, ajuste la date de début

                                

                                

                                    
Avantage : Préserve la logique métier tout en anonymisant

                                

                                

                                    
Attention : Nécessite maintenance des fonctions custom
